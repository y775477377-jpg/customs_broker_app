# دليل المخلص الجمركي اليمني — تطبيق الموبايل (Flutter / أندرويد)

⚠️ **تنبيه مهم وصريح قبل كل شيء:** بيئة التنفيذ التي كتبتُ فيها هذا الكود **لا تملك Flutter/Dart SDK ولا اتصال إنترنت**، لذا لم أستطع تشغيل `flutter pub get` ولا `flutter analyze` ولا `flutter build` هنا للتحقق الآلي من الكود، كما فعلت مع الـ Backend (حيث تحققت فعلياً بـ `node --check` و27 اختباراً ناجحاً). كل ملفات Dart كُتبت ومُراجعت يدوياً بعناية، لكن **يجب عليك تشغيل `flutter analyze` على جهازك أولاً** قبل الاعتماد على الكود، وإخباري بأي خطأ يظهر لإصلاحه فوراً.

## الخطوات المطلوبة على جهازك لتشغيل المشروع

```bash
# 1) تأكد من تثبيت Flutter SDK (قناة stable)
flutter doctor

# 2) داخل مجلد المشروع (الذي يحتوي هذا الملف)، أنشئ مجلدات المنصات الناقصة
#    (android/ios/إلخ) — هذا الأمر آمن على مشروع Dart-only موجود مسبقاً،
#    لن يحذف lib/ أو pubspec.yaml، فقط يضيف ما ينقص:
flutter create --platforms=android,ios --org com.yourcompany .

# 3) ثبّت الحزم
flutter pub get

# 4) تحقق الآن من الكود فعلياً (خطوة إلزامية قبل أي بناء):
flutter analyze

# 5) شغّل على محاكي أندرويد أو جهاز حقيقي متصل
flutter run
```

## ربط التطبيق بالـ Backend
افتراضياً، `lib/core/api_client.dart` يستخدم `http://10.0.2.2:4000` وهو العنوان الذي يُترجم لـ`localhost` الخاص بجهاز التطوير **من داخل محاكي أندرويد فقط**. لتغييره (جهاز حقيقي، أو خادم إنتاج):

```bash
flutter run --dart-define=API_BASE_URL=https://api.yourdomain.com
```

أو عدّل القيمة الافتراضية مباشرة في `ApiConfig.baseUrl`.

## ما تم تنفيذه في هذه النسخة
- **تسجيل الدخول**: مستخدمو المكاتب (`office_code` + بريد + كلمة مرور) وSuper Admin (بريد + كلمة مرور فقط)، مطابق تماماً لتصميم `docs/AUTH_SESSIONS.md` في الـ Backend.
- **إدارة الجلسة**: تخزين آمن للتوكنات (Keystore/Keychain عبر `flutter_secure_storage`)، تجديد تلقائي للـ Access Token عند 401، وتوجيه فوري لشاشة الدخول عند فشل التجديد (Reuse Detection من الخادم).
- **الإقرارات الجمركية**: قائمة مع فلترة حسب الحالة، إنشاء إقرار جديد (مع اختيار التاجر عبر بحث)، تفاصيل الإقرار مع الأصناف، إضافة/حذف صنف (مع بحث تعرفة اقتراحي)، **تشغيل الحساب**، **الاعتماد**، **إعادة الفتح (Amendment)** — تعكس دورة الحياة الصارمة `draft → calculated → approved` من الـ Backend حرفياً بما في ذلك رسائل `STALE_CALCULATION` و`DECLARATION_LOCKED`.
- **التجار**: قائمة مع بحث، إنشاء تاجر جديد.
- **معالجة أخطاء موحدة**: `lib/core/api_exception.dart` يحوّل كل رمز خطأ من الـ Backend (`SUBSCRIPTION_EXPIRED`, `TENANT_SUSPENDED`, `AMBIGUOUS_TARIFF_RULE`, ...) إلى رسالة عربية واضحة للمستخدم.
- دعم عربي RTL كامل (`Directionality` + `flutter_localizations`).

## ما لم يُنفَّذ بعد (بصراحة)
- شاشات **Super Admin** (إدارة المكاتب، الباقات، قواعد التعرفة، الاعتماد...) — هذه مخطط لها أصلاً كـ **Web Admin Dashboard منفصل** حسب وثيقة التصميم الأصلية، وليست جزءاً من تطبيق الموبايل الموجّه لمكاتب التخليص.
- النظام المحاسبي (سندات القبض/الصرف، كشف الحساب) والتقارير — Phase 4/5 حسب الخطة الأصلية في `docs/PHASE3_ARCHITECTURE.md` بالـ Backend.
- رفع المستندات (PDF/صور) وربطها بالإقرار.
- اختبارات Widget/Integration لتطبيق Flutter نفسه (`flutter test`) — لم تُكتب بعد؛ أولوية قادمة.
- أيقونة التطبيق والشعار المخصصان (Placeholder افتراضي من Flutter حالياً).

## android/ios
لم تُنشأ مجلدات `android/` و`ios/` في هذه الحزمة لأن إنشاءها يدوياً بدون Flutter SDK لتوليدها وفحصها عالي الخطورة (ملفات Gradle/Xcode دقيقة جداً وعرضة للأخطاء بدون أداة رسمية تنشئها وتتحقق منها). الخطوة 2 أعلاه (`flutter create --platforms=android,ios .`) تُنشئها تلقائياً وبشكل موثوق باستخدام أداة Flutter الرسمية نفسها.
